# olist-ecommerse-analysis
# Olist E-commerce Data Analysis

## 📌 Project Overview
This project is an exploratory data analysis (EDA) of the Brazilian e-commerce dataset from Olist.

The goal is to understand business performance, customer behavior, payment patterns, and delivery efficiency using SQL and Python (Pandas).

---

## 📊 Dataset Description
The dataset includes information about:
- orders
- customers
- order items
- payments
- products

---

## 🛠 Tools Used
- Python
- Pandas
- NumPy
- Matplotlib
- SQL
- Jupyter Notebook

---

## 🎯 Key Business Questions
- How many orders and customers are in the system?
- What is the total GMV of the platform?
- Which payment methods are most used?
- Which regions generate the most revenue?
- What is the repeat purchase rate?
- How long does delivery take?
- What is the delay rate of deliveries?

---

## 📈 Key Insights

-Repeat purchase rate: 3.12% → low retention
-Avg delivery time: 12.09 days → logistics bottleneck
-Late delivery rate: 7.87%
-SP generates ~40%+ of revenue → high dependency risk
- Total orders analyzed: ~99,000
- Unique customers: ~96,000
- Late deliveries: ~7.9%
- ~90% of orders contain only one item
- Peak sales observed in November (seasonality effect)

---

## 💡 Business Conclusions

- The platform is highly dependent on first-time buyers (low retention)
- Delivery performance is generally good, but delays still exist
- Sales are strongly seasonal with peaks in November
- Most revenue comes from a small number of regions (especially SP state)
